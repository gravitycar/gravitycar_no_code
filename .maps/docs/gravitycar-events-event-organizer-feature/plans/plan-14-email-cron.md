# Implementation Plan: Email Reminder Cron Job + PHPMailer

## Spec Context

The Email Reminder Cron Job is the background process that drives the entire email reminder pipeline. It has two responsibilities: (1) process pending reminders by creating email queue entries for all invitees, and (2) process the email queue by sending emails via PHPMailer with retry/exponential backoff (3 attempts: 5min, 30min, 2hrs). Reminder emails include event details and an ICS attachment generated by the IcsGeneratorService (from item 13).

- **Catalog item**: 14 - Email Reminder Cron Job + PHPMailer
- **Specification section**: Email Reminders section (Background process), Email Queue section
- **Acceptance criteria addressed**: AC-11 (pending reminders sent to all invitees exactly once, NULL remind_at skipped)

## Dependencies

- **Blocked by**: Item 7 (Event_Reminders Model -- provides reminder records and `listByEventId`), Item 8 (Email_Queue Model -- provides queue records, `findPendingEmails`, `markAsSent`, `markAsFailedOrRetry`), Item 13 (ICS Export -- provides `IcsGeneratorService` for ICS attachments)
- **Uses**: `Gravitycar\Models\event_reminders\EventReminders`, `Gravitycar\Models\email_queue\EmailQueue`, `Gravitycar\Services\IcsGeneratorService`, `Gravitycar\Core\Config`, `Gravitycar\Factories\ModelFactory`, `PHPMailer\PHPMailer\PHPMailer`

**Public Methods:**

#### `processReminders(): array`

Finds all pending reminders whose `remind_at` has passed, then creates email queue entries for each invitee. Marks the reminder as "sent" after all queue entries are created. Returns a summary array.

```php
public function processReminders(): array
{
    $reminders = $this->findDueReminders();
    $processed = 0;
    $emailsQueued = 0;

    foreach ($reminders as $reminderData) {
        $queued = $this->queueEmailsForReminder($reminderData);
        $this->markReminderAsSent($reminderData['id']);
        $emailsQueued += $queued;
        $processed++;
    }

    $this->logger->info('Processed reminders', [
        'reminders_processed' => $processed,
        'emails_queued' => $emailsQueued,
    ]);

    return [
        'reminders_processed' => $processed,
        'emails_queued' => $emailsQueued,
    ];
}
```

#### `processEmailQueue(): array`

Finds all pending emails whose `send_at` has passed and sends them via PHPMailer. Handles success/failure with retry logic delegated to `EmailQueue::markAsFailedOrRetry()`. Returns a summary array.

```php
public function processEmailQueue(): array
{
    $emailQueueModel = $this->modelFactory->create('Email_Queue');
    $pendingEmails = $emailQueueModel->findPendingEmails();
    $batchSize = (int) $this->config->get('email_queue_batch_size', 20);

    $sent = 0;
    $failed = 0;

    $pendingEmails = array_slice($pendingEmails, 0, $batchSize);

    foreach ($pendingEmails as $emailData) {
        $success = $this->sendEmail($emailData);
        if ($success) {
            $emailQueueModel->markAsSent($emailData['id']);
            $sent++;
        } else {
            $failed++;
            // markAsFailedOrRetry already called inside sendEmail on failure
        }
    }

    $this->logger->info('Processed email queue', [
        'sent' => $sent,
        'failed' => $failed,
    ]);

    return ['sent' => $sent, 'failed' => $failed];
}
```

#### `run(): array`

Main entry point called by the cron script. Runs both phases sequentially.

```php
public function run(): array
{
    $this->logger->info('Email reminder cron job started');

    $reminderResults = $this->processReminders();
    $emailResults = $this->processEmailQueue();

    $this->logger->info('Email reminder cron job completed');

    return [
        'reminders' => $reminderResults,
        'emails' => $emailResults,
    ];
}
```

**Private Methods:**

#### `findDueReminders(): array`

Queries Event_Reminders for records where `status = 'pending'`, `remind_at IS NOT NULL`, and `remind_at <= NOW()`.

```php
private function findDueReminders(): array
{
    $reminderModel = $this->modelFactory->new('Event_Reminders');
    $now = (new \DateTimeImmutable('now', new \DateTimeZone('UTC')))
        ->format('Y-m-d H:i:s');

    return $reminderModel->findRaw(
        ['status' => self::REMINDER_STATUS_PENDING],
        [],
        [
            'where' => ['remind_at IS NOT NULL', 'remind_at <= :now'],
            'params' => ['now' => $now],
        ]
    );
}
```

#### `queueEmailsForReminder(array $reminderData): int`

Fetches the event details and all invited users, then creates one Email_Queue entry per invitee. Attaches ICS content if the event has an accepted_date. Returns the count of emails queued.

```php
private function queueEmailsForReminder(array $reminderData): int
{
    $eventId = $reminderData['event_id'];
    $eventsModel = $this->modelFactory->create('Events');
    $eventData = $eventsModel->read($eventId);

    if (empty($eventData)) {
        $this->logger->warning('Event not found for reminder', [
            'reminder_id' => $reminderData['id'],
            'event_id' => $eventId,
        ]);
        return 0;
    }

    $invitees = $this->getEventInvitees($eventId);
    if (empty($invitees)) {
        $this->logger->info('No invitees for event', ['event_id' => $eventId]);
        return 0;
    }

    $emailBody = $this->buildReminderEmailBody($eventData);
    $icsContent = $this->generateIcsAttachment($eventData);
    $subject = 'Reminder: ' . ($eventData['name'] ?? 'Event');
    $now = (new \DateTimeImmutable('now', new \DateTimeZone('UTC')))
        ->format('Y-m-d H:i:s');

    $queued = 0;

    foreach ($invitees as $invitee) {
        // Create a new model instance per iteration to avoid stale state from previous create()
        $emailQueueModel = $this->modelFactory->create('Email_Queue');
        $emailQueueModel->set('recipient_email', $invitee['email']);
        $emailQueueModel->set('recipient_user_id', $invitee['id']);
        $emailQueueModel->set('subject', $subject);
        $emailQueueModel->set('body', $emailBody);
        $emailQueueModel->set('status', self::EMAIL_STATUS_PENDING);
        $emailQueueModel->set('send_at', $now);
        $emailQueueModel->set('related_event_id', $eventId);
        $emailQueueModel->set('related_reminder_id', $reminderData['id']);

        if ($emailQueueModel->create()) {
            $queued++;
        }
    }

    return $queued;
}
```

#### `getEventInvitees(string $eventId): array`

Uses the framework's relationship API to get all invited users with their email addresses.

```php
private function getEventInvitees(string $eventId): array
{
    $eventsModel = $this->modelFactory->create('Events');
    $eventsModel->findById($eventId);

    $relatedUsers = $eventsModel->getRelatedModels('events_users_invitations');

    $invitees = [];
    foreach ($relatedUsers as $userModel) {
        $invitees[] = [
            'id' => $userModel->get('id'),
            'email' => $userModel->get('email'),
        ];
    }

    return $invitees;
}
```

**Key decision**: Uses `ModelBase::getRelatedModels()` instead of raw SQL with JOIN against the `events_users_invitations` table. The `RelationshipBase` class handles all table/column naming internally.

#### `buildReminderEmailBody(array $eventData): string`

Builds an HTML email body with event name, accepted date, location, and description.

```php
private function buildReminderEmailBody(array $eventData): string
{
    $name = htmlspecialchars($eventData['name'] ?? '', ENT_QUOTES, 'UTF-8');
    $location = htmlspecialchars($eventData['location'] ?? '', ENT_QUOTES, 'UTF-8');
    $description = htmlspecialchars(
        $eventData['description'] ?? '',
        ENT_QUOTES,
        'UTF-8'
    );
    $acceptedDate = $eventData['accepted_date'] ?? 'TBD';

    return <<<HTML
    <h2>Event Reminder: {$name}</h2>
    <p><strong>Date:</strong> {$acceptedDate}</p>
    <p><strong>Location:</strong> {$location}</p>
    <p>{$description}</p>
    <p>An ICS calendar file is attached for your convenience.</p>
    HTML;
}
```

#### `generateIcsAttachment(array $eventData): ?string`

Returns ICS content string if the event has an accepted_date, or null otherwise. Reuses `IcsGeneratorService` from item 13.

```php
private function generateIcsAttachment(array $eventData): ?string
{
    if (empty($eventData['accepted_date'])) {
        return null;
    }

    try {
        return $this->icsGeneratorService->generateIcsContent($eventData);
    } catch (\Exception $e) {
        $this->logger->error('Failed to generate ICS attachment', [
            'event_id' => $eventData['id'] ?? 'unknown',
            'error' => $e->getMessage(),
        ]);
        return null;
    }
}
```

#### `sendEmail(array $emailData): bool`

Sends a single email via PHPMailer. Attaches ICS content if the related event has an accepted_date. On failure, calls `EmailQueue::markAsFailedOrRetry()`.

```php
private function sendEmail(array $emailData): bool
{
    $mailer = $this->createMailer();

    try {
        $mailer->addAddress($emailData['recipient_email']);
        $mailer->Subject = $emailData['subject'];
        $mailer->isHTML(true);
        $mailer->Body = $emailData['body'];
        $mailer->AltBody = strip_tags($emailData['body']);

        $icsContent = $this->getIcsForEmail($emailData);
        if ($icsContent !== null) {
            $mailer->addStringAttachment(
                $icsContent,
                'event.ics',
                'base64',
                'text/calendar'
            );
        }

        $mailer->send();

        $this->logger->info('Email sent successfully', [
            'email_id' => $emailData['id'],
            'recipient' => $emailData['recipient_email'],
        ]);

        return true;
    } catch (\Exception $e) {
        $this->logger->error('Email send failed', [
            'email_id' => $emailData['id'],
            'recipient' => $emailData['recipient_email'],
            'error' => $e->getMessage(),
        ]);

        $emailQueueModel = $this->modelFactory->create('Email_Queue');
        $emailQueueModel->markAsFailedOrRetry(
            $emailData['id'],
            $e->getMessage()
        );

        return false;
    }
}
```

#### `getIcsForEmail(array $emailData): ?string`

Loads the related event data and generates ICS content if the event has an accepted_date.

```php
private function getIcsForEmail(array $emailData): ?string
{
    $eventId = $emailData['related_event_id'] ?? null;
    if (empty($eventId)) {
        return null;
    }

    $eventsModel = $this->modelFactory->create('Events');
    $eventData = $eventsModel->read($eventId);
    if (empty($eventData)) {
        return null;
    }

    return $this->generateIcsAttachment($eventData);
}
```

#### `createMailer(): PHPMailer`

Creates and configures a PHPMailer instance from config values.

```php
protected function createMailer(): PHPMailer
{
    $mailer = new PHPMailer(true);
    $mailer->isSMTP();
    $mailer->Host = $this->config->get('smtp_host');
    $mailer->Port = (int) $this->config->get('smtp_port', 587);
    $mailer->SMTPAuth = true;
    $mailer->Username = $this->config->get('smtp_username');
    $mailer->Password = $this->config->get('smtp_password');
    $mailer->SMTPSecure = $this->config->get('smtp_encryption', 'tls');
    $mailer->setFrom(
        $this->config->get('email_from_address', 'noreply@gravitycar.com'),
        $this->config->get('email_from_name', 'Gravitycar Events')
    );
    $mailer->CharSet = 'UTF-8';

    return $mailer;
}
```

#### `markReminderAsSent(string $reminderId): void`

Updates the reminder record to status="sent" and sets sent_at.

```php
private function markReminderAsSent(string $reminderId): void
{
    $reminderModel = $this->modelFactory->create('Event_Reminders');
    $data = $reminderModel->read($reminderId);
    if (empty($data)) {
        return;
    }

    foreach ($data as $field => $value) {
        if ($reminderModel->hasField($field)) {
            $reminderModel->set($field, $value);
        }
    }

    $now = (new \DateTimeImmutable('now', new \DateTimeZone('UTC')))
        ->format('Y-m-d H:i:s');
    $reminderModel->set('status', self::REMINDER_STATUS_SENT);
    $reminderModel->set('sent_at', $now);
    $reminderModel->update();
}
```

### 4. Cron Entry Point

**File**: `bin/process_email_queue.php`

This is the CLI script invoked by crontab. It bootstraps the framework, creates the service, and calls `run()`.

```php
<?php
declare(strict_types=1);

require_once __DIR__ . '/../vendor/autoload.php';

use Gravitycar\Core\Config;
use Gravitycar\Factories\ModelFactory;
use Gravitycar\Services\IcsGeneratorService;
use Gravitycar\Services\EmailReminderService;
use Monolog\Logger;
use Monolog\Handler\StreamHandler;

$logger = new Logger('email-cron');
$logger->pushHandler(new StreamHandler(
    __DIR__ . '/../logs/email_cron.log',
    Logger::INFO
));

try {
    $config = new Config();
    $modelFactory = new ModelFactory();
    $icsService = new IcsGeneratorService($logger, $config);

    $service = new EmailReminderService(
        $logger,
        $config,
        $modelFactory,
        $icsService
    );

    $results = $service->run();

    $logger->info('Cron job finished', $results);
} catch (\Throwable $e) {
    $logger->critical('Cron job failed with exception', [
        'error' => $e->getMessage(),
        'trace' => $e->getTraceAsString(),
    ]);
    exit(1);
}
```

**Crontab entry** (runs every minute):

```
* * * * * /usr/bin/php /path/to/gravitycar/bin/process_email_queue.php >> /path/to/gravitycar/logs/email_cron_stdout.log 2>&1
```

## Error Handling

- **Event not found for reminder**: Logs warning, skips that reminder (returns 0 queued emails), does NOT mark as sent.
- **No invitees for event**: Logs info, marks reminder as sent (nothing to send is not an error).
- **ICS generation failure**: Logs error, sends email without ICS attachment (graceful degradation).
- **PHPMailer send failure**: Logs error, calls `EmailQueue::markAsFailedOrRetry()` which handles retry scheduling and permanent failure after 3 attempts.
- **SMTP config missing**: `createMailer()` will throw when PHPMailer cannot connect; caught in `sendEmail()` and handled as a send failure with retry.
- **Cron script top-level exception**: Caught in `bin/process_email_queue.php`, logged as critical, exits with code 1.
- **Database connection errors**: DBAL exceptions propagate up to the cron script's top-level catch.

### Key Scenario: Full Cron Run with Mixed Results

**Setup**:
- 1 due reminder (remind_at in past, status=pending) for event "Game Night" with 2 invitees
- 1 existing pending email in queue (from previous run) ready to send
- Mock PHPMailer to succeed on all sends

**Action**: Call `run()`

**Expected**:
- `processReminders()`: 1 reminder processed, 2 emails queued
- `processEmailQueue()`: 3 emails sent (1 existing + 2 newly queued), 0 failed
- Reminder status updated to "sent" with sent_at set
- All 3 email queue records have status "sent" with sent_at set

### Key Scenario: Duplicate Prevention

**Setup**:
- 1 reminder already with status='sent'
- Same reminder's remind_at is still in the past

**Action**: Call `processReminders()`

**Expected**: The reminder is NOT returned by `findDueReminders()` because the query filters on `status = 'pending'`. Returns reminders_processed=0.

## Notes

- **Batch size**: The `email_queue_batch_size` config limits how many emails are sent per cron invocation. This prevents overwhelming the SMTP server and keeps each cron run fast. Default is 20.
- **Cron frequency**: The script is designed to run every minute via crontab. Each run processes due reminders first (creating queue entries), then processes the email queue. This two-phase approach means newly queued emails from a reminder can be sent in the same cron run.
- **createMailer() is protected**: This allows test subclasses to override it and inject a mock PHPMailer without needing to pass PHPMailer as a constructor dependency.
- **ICS attachment is best-effort**: If ICS generation fails for any reason, the email is still sent without the attachment. This prevents ICS issues from blocking reminder delivery.
- The framework's relationship API (`getRelatedModels()`) handles join table column naming internally, so no hardcoded column names are needed for fetching invitees.
- **Model instance per loop iteration**: In `queueEmailsForReminder()`, a new `Email_Queue` model instance is created per loop iteration via `$this->modelFactory->create('Email_Queue')`. This avoids a bug where reusing the same model instance would carry stale state (e.g., the `id` from a previous `create()` call) into subsequent iterations.
- **No lock file mechanism**: For simplicity, no file-based lock is implemented to prevent overlapping cron runs. The query-then-update pattern with status fields provides sufficient protection against duplicate processing. If race conditions become an issue under heavy load, a lock file or `SELECT ... FOR UPDATE` can be added later.
